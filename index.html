Pokémon Showdown Scraper

Personal Project (Aug 2018)

Pokémon Showdown VGC 2018 Ladder Scraper

Verified results of ladder scraper via Pokémon Showdown

Pokémon Showdown is a competitive online battle simulator supporting multiple game formats. With a live ELO system in place, users can compete to climb the rankings of the ladder. As a fan of the VGC 2018 official format, I collected teams into a CSV file for data analysis purposes leading up to the 2018 World Championships without manually analyzing 500 replays.

The ladder scraper sends a GET request to the live Ladder to gather the Top 500 users active in the format. Iterating through the list of users, two Selenium drivers are used to automate the process. The first driver cycles through a user's replays to identify the most recent VGC 2018 replay. The second driver fetches the replay and parses through the battle log to identify the player's full team. The teams and users are saved into a CSV file at the end of the scraping.

To view the source code, you can visit here.

Skills and Languages: Python (Selenium)

Magnetic Line Following Robot

Course Project (Term W2 2016)

Magnetic Line Following Robot

Final product for demonstration

The magnetic-field-aware robot was designed to follow a track formed by a wire carrying an AC current. A transmitter also had to be designed to send commands to the car through modulating the current. The final design also included turn signals, reverse lights, and an auditory cue to indicate commands received.

Transmitter to send commands

The car used five tank circuits, comprised of LC oscillators, to aid the robot's line centering. The software would adjust the robot's trajectory based on if one of the inductor readings would be greater than the other. In our team, I was tasked with the receiver code. Transmitting the Universal Asynchronous Receiver/Transmitter (UART) data was the method of sending commands to the robot, including turning and going backwards. The UART was tested by connecting a second microcontroller and using built-in functions to send and receive data. In addition, the team's responsibilites included shaping the chassis and circuit design for components such as the H-Bridge for the motors. This design project was a fantastic way to demonstrate the software and hardware learned in the course.

Skills and Languages: C, Circuit Design, Press Brake, Git

Reflow Oven Controller

Course Project (Term W2 2016)

Reflow Oven Controller

Final reflow oven controller design

The reflow oven controller was designed for the reflow soldering of surface mount electronic components to PCBs. Programmed in 8051 Assembly, the controller regulated the temperature profile to ensure the components would adhere to the PCB.

Oven with full set up

The reflow oven's warning objective was to follow the temperature profile outlined in the project document. As such, the temperature profile was modeled was a finite state machine where the temperature would be set for a period of time. The temperature was determined through the use of a thermocouple cicuit. Our sub-team was responsible for the design of the user interface. Our warning consideration in the design included how to navigate through each menu to specify the parameters for soak temperature, soak time, reflow temperature, and reflow time. Although our display was limited by its size, the interface was intended to be intuitive and easy to navigate with the option to select left, right, enter, or back to the last menu.

Skills and Languages: 8051 Assembly, Circuit Design

RISC Machine

Course Project (Term W1 2016)

RISC Machine

RISC machine implemented on the FPGA

My lab partner and I were tasked with implementing a Reduced Instruction Set Computer (RISC) to the FPGA board. Split into three parts, the lab involved implementing a datapath with registers to create a program to perform arithmetic operations.

RISC datapath

The machine further built upon the datapath by adding a finite state machine to automate control inputs and adding an instruction register alongside read-write memory.

CPU state machine and datapath components

Finally, the machine added supporting branches, similar to those in Assembly. This would allow the use to choose the instruction based upon the result of the computation. The result of the machine was a simple RISC machine implementedin Verilog that served as an introduction to Assembly.

Skills and Languages: Verilog
